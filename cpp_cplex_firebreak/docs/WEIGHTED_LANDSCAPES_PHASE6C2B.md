# Weighted Landscapes Phase 6C2B: Initial and Fractional Combinatorial Cuts

Phase 6C2B audits and enables the existing FPP combinatorial cut mechanisms
outside integer-incumbent lazy separation for arbitrary positive landscape
weights. The scope is limited to:

- binary initial combinatorial cuts generated before CPLEX optimization;
- fractional path-activation user cuts generated in CPLEX relaxation callbacks;
- diagnostics for the absence of a dedicated combinatorial root-only mechanism.

It does not add scenario sampling, eta-desc ordering, restricted-candidate
combinatorial Benders, DPV, Static-DPV, manifests, or batch integration.

## Audited Mechanisms

```text
mechanism        option/context                         classification
initial cuts     --combinatorial-benders-initial-cuts   globally valid static cuts
fractional cuts  --combinatorial-benders-separate-fractional / relaxation callback
                                                       convex-hull-valid user cuts
root cuts        --use-root-user-cuts                   LP-dual Benders, not combinatorial
```

There is no separate combinatorial root-only separator in the repository.
`--use-root-user-cuts` invokes LP-dual Benders root user cuts through
`FppBranchBendersRootUserCutCallback`. When combinatorial Benders is enabled,
that LP-dual root path remains separate and is still rejected for
non-homogeneous combinatorial exact mode.

## Initial Cuts

Initial cuts are generated in `FppBranchBendersSolver::solve` before the MIP
solve:

1. Build `FppCombinatorialBendersSeparator`.
2. Compute `greedyInitialSolution()`, a deterministic binary feasible firebreak
   vector.
3. Call `initialCutsFromSolution()`.
4. Add one static master cut per scenario unless its weighted cut signature was
   already inserted.
5. Add the same binary vector as a MIP start.

The cut family is the Phase 6C1 weighted path-activation cut, with Phase 6C2A
path-dedup lifting allowed for `heuristic` and `posterior`:

```text
eta_s >= gamma_s + sum_i alpha_i y_i
gamma_s = sum_{k in A_s(ybar)} w_k
alpha_i = - sum_{k: i on selected path P_{s,k}} w_k
```

The generating solution is binary. Constants and coefficients are in weighted
destination-loss units; candidate weights are not used as protected-value
proxies. The ignition/root always contributes its own weight to `gamma_s` when
burned and is not blocked by selecting it.

Because each path inequality is valid for every binary feasible solution, the
static initial cut is globally valid for the integer epigraph. It is tight at
the generating binary solution. With lifted modes, it uses only the validated
Phase 6C2A path-dedup transformation.

## Fractional User Cuts

Fractional cuts are generated by `FppCombinatorialBranchBendersCallback` in
CPLEX generic `Relaxation` context when
`combinatorial_benders_separate_fractional=true`.

At a relaxation point `(ybar, eta)` the separator:

1. reads the complete fractional eligible-candidate vector;
2. computes shortest directed activation paths with eligible-node costs
   `ybar_i`;
3. marks destinations with path distance `< 1`;
4. builds one affine path inequality per active destination;
5. sums the inequalities into one scenario cut;
6. adds violated cuts through `context.addUserCut`.

For a destination `k` and selected path `P_{s,k}`, the per-destination
inequality is:

```text
w_k * (1 - sum_{i in P_{s,k}} y_i) <= w_k * x_k
```

For binary `y`, if no candidate on `P_{s,k}` is selected, that path remains
open and `k` burns. If at least one candidate on the path is selected, the left
side is non-positive and is bounded by the nonnegative burn indicator. The
inequality is affine, so binary validity extends to every convex combination of
binary epigraph points. Therefore the summed weighted cut is valid as a global
MIP user cut over the convex hull of feasible binary recourse epigraph points.

Fractional lifting is not enabled. If `lift_mode=heuristic` or `posterior` is
requested, fractional separation falls back to the non-lifted path cut and
records the fallback. Integer candidate cuts in the same solve still use the
requested Phase 6C2A lifting mode.

## Root Policy

Phase 6C2B does not add root-only combinatorial cut rounds. Fractional
combinatorial cuts are relaxation-context user cuts and may be separated where
CPLEX invokes the relaxation callback. Root-only LP-dual Benders cuts retain
their own diagnostics and remain mutually exclusive with non-homogeneous
weighted combinatorial exact mode.

## Supported Non-Homogeneous Configuration

For non-homogeneous maps, Phase 6C2B allows:

```text
enabled=true
lift_mode=none|heuristic|posterior
scenario_order=eta-asc
cut_sampling_ratio=1
initial_cuts=true|false
separate_fractional=true|false
```

The following remain rejected:

- `cut_sampling_ratio < 1`;
- `scenario_order=eta-desc`;
- LP-dual `--use-root-user-cuts` combined with combinatorial Benders;
- LLBI/projected LLBI combined with combinatorial Benders;
- global dominance or conditional zero-benefit fixing combined with
  combinatorial Benders;
- restricted-candidate combinatorial methods;
- DPV and Static-DPV combinatorial variants.
